# Creating a user with an excessively large name field is not rejected

## Description
The user creation endpoint does not validate the length of the name field. When a name field containing a very large string is submitted, the API is expected to reject the request with a 400 status code and an error message stating the name is too large. Instead, the API accepts the request and echoes the oversized value back without any validation error, allowing arbitrarily large data to be stored.

## Steps to Reproduce
1. Send a create user request where the name field contains an extremely long string (tens of thousands of characters), along with a valid email and age.
2. Observe the response status code and body.

## Actual Outcome
Based on Sample Payload below:
```Json
{
    "name": "<string with 74951 characters>",
    "email": "Alyce14@yahoo.com",
    "age": 7
}
```

## Expected Outcome
The API should return a 400 status code with a body indicating the name field is too large, for example:
```json
{
    "message": "name is too large"
}
```

## Sample Payload
POST /users
```Json
{
    "name": "<string with 74951 characters>",
    "email": "Alyce14@yahoo.com",
    "age": 7
}
```
