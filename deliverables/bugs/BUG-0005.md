# Creating a user with an invalid email format is not rejected

## Description
The user creation endpoint does not validate the format of the email field. When a value that does not conform to a valid email format is submitted, the API is expected to reject the request with a 400 status code and an error message stating the email format is invalid. Instead, the API accepts the request and echoes the invalid value back without any validation error.

## Steps to Reproduce
1. Send a create user request with a name, age, and an email value that is not a valid email format, for example a string without an "@" symbol.
2. Observe the response status code and body.

## Actual Outcome
Based on Sample Payload below:
```json
{
    "name": "Christine Erdman",
    "email": "notanemail.com",
    "age": 52
}
```

## Expected Outcome
The API should return a 400 status code with a body indicating the email format is invalid, for example:
```json
{
    "message": "Invalid email format"
}
```

## Sample Payload
POST /users
```json
{
    "name": "Christine Erdman",
    "email": "notanemail.com",
    "age": 52
}
```

## Environments
dev, prod
