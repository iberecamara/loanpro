# Updating a user with an excessively large name field is not rejected

## Description
The update user endpoint does not validate the length of the name field. When an existing user is updated with a name field containing a very large string, the API is expected to reject the request with a 400 status code and an error message stating the name is too large. Instead, the API accepts the request without returning any validation error.

## Steps to Reproduce
1. Seed a user in the system so an existing record is available to update.
2. Send an update request for that user's email where the name field contains an extremely long string (tens of thousands of characters), along with a valid email and age.
3. Observe the response status code and body.

## Actual Outcome
Based on Sample Payload below:
```json
{
    "name": "<string with 74951 characters>",
    "email": "Freeda_Stamm62@hotmail.com",
    "age": 83
}
```

## Expected Outcome
The API should return a 400 status code with a body indicating the name field is too large, for example:
```json
{
    "message": "name is too large"
}
```

## Sample Payload
`PUT /users/Khalid18@yahoo.com`
```json
{
    "name": "<string with 74951 characters>",
    "email": "Freeda_Stamm62@hotmail.com",
    "age": 83
}
```
