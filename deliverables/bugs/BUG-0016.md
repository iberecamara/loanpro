# Deleting an existing user returns 401 instead of 204, preventing verification that the user is removed from Get All Users

## Description
This scenario verifies that once a user is deleted, that user no longer appears in the Get All Users response. The delete step itself is expected to succeed with a 204 status code using valid credentials. Instead, the delete request fails with a 401 status code, which prevents the test from reaching the verification step that checks the user is absent from the Get All Users list.

## Steps to Reproduce
1. Seed a user in the system so an existing record is available to delete.
2. Send a DELETE request to that user's email endpoint using valid credentials.
3. Observe the response status code of the delete request.

## Actual Outcome
The API returns a 401 status code for the delete request, so the user is not removed and the subsequent Get All Users verification cannot be performed.

## Expected Outcome
The API should return a 204 status code for the delete request, after which the deleted user should no longer appear in the Get All Users response.

## Sample Payload
`DELETE /users/Aubree24@gmail.com`
```
Authorization: <valid token>
```
