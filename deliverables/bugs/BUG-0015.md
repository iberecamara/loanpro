# Deleting a non-existing user without an Authorization header returns 404 instead of 401

## Description
When a delete request is sent for an email that does not correspond to any existing user, and no Authorization header is provided, the API is expected to reject the request with a 401 status code before checking whether the user exists. Instead, the API returns a 404 status code with a "User not found" message, indicating the existence check is being performed before the authorization check.

## Steps to Reproduce
1. Send a DELETE request to an email endpoint that does not correspond to any existing user, without including an Authorization header.
2. Observe the response status code and body.

## Actual Outcome
The API returns a 404 status code with the body:
```json
{
    "message": "User not found"
}
```

## Expected Outcome
The API should return a 401 status code with a body indicating authentication is required, for example:
```json
{
    "message": "Authentication required"
}
```

## Sample Payload
`DELETE /users/non.existing.emailOlaf_Denesik99@gmail.com`

(no Authorization header included)
