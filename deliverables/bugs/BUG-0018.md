# Updating a user does not persist the updated data

## Description
When using the Update User feature, the API responds with the user updated data, but when searching for the user afterwards the data is not updated. This indicates that while the response uses the updated data but that is not persisted in the database.

## Steps to Reproduce
1. Seed a user in the system so an existing record is available.
2. Send a PU request to that user's email endpoint with updated name, email, and age values.
3. Observe the response status code and body.
4. Search the user with it's email for the update check.
5. Observe the response status code and body.

## Actual Outcome
The API returns the previous user data:
```json
{
    "name": "Teri Casper-Ratke",
    "email": "Marlen.Cartwright@hotmail.com",
    "age": 120
}
```

## Expected Outcome
The API returns the updated user data:
```json
{
    "name": "Different Name",
    "email": "Marlen.Cartwright@hotmail.com",
    "age": 10
}
```

## Sample Payload
`POST /users`
```json
{
    "name": "Teri Casper-Ratke",
    "email": "Marlen.Cartwright@hotmail.com",
    "age": 120
}
```
`PUT /users/Kenya74@gmail.com`
```json
{
    "name": "Different Name",
    "email": "Marlen.Cartwright@hotmail.com",
    "age": 10
}
```
