# Creating a user with a duplicated email returns 500 instead of 409

## Description
When a user is created with an email that already exists in the system, the API should reject the request with a 409 Conflict response indicating the email is already in use. Instead, the request fails with a 500 Internal Server Error, meaning the duplicate email check is not being handled gracefully by the API.

## Steps to Reproduce
1. Create a new user with a unique name, email, and age.
2. Send a second create user request using the same email as the previous user, with a different name and age.
3. Observe the response status code and body for the second request.

## Actual Outcome
The second request returns a 500 status code with the body:
```json
{
    "message": "Internal server error"
}
```

## Expected Outcome
The second request should return a 409 status code with a body indicating the email is already in use, for example:
```json
{
    "message": "Email already exists"
}
```

## Sample Payload
First request:

POST /users
```json
{
    "name": "Art Koss",
    "email": "Reba_Parisian85@yahoo.com",
    "age": 82
}
```

Followed by:

POST /users
```json
{
    "name": "Art Koss",
    "email": "Reba_Parisian85@yahoo.com",
    "age": 82
}
```
