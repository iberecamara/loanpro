# Deleting an existing user without an Authorization header succeeds instead of being rejected

## Description
When a delete request is sent for an existing user without providing an Authorization header, the API is expected to reject the request with a 401 status code and an "Authentication required" message. Instead, the API processes the request and deletes the user, returning a 204 status code, meaning the delete endpoint does not enforce authentication.

## Steps to Reproduce
1. Seed a user in the system so an existing record is available to delete.
2. Send a DELETE request to that user's email endpoint without including an Authorization header.
3. Observe the response status code.

## Actual Outcome
The API returns a 204 status code and the user is deleted, even though no Authorization header was provided.

## Expected Outcome
The API should return a 401 status code with a body indicating authentication is required, for example:
```json
{
    "message": "Authentication required"
}
```

## Sample Payload
`DELETE /users/Luke33@gmail.com`
```
(no Authorization header included)
```
