# DELETE on the users collection returns 500 instead of 405

## Description
Sending a DELETE request to the users collection endpoint (`/users`) is not a supported operation, since deletion only applies to a single user by email. The API is expected to reject this request with a 405 Method Not Allowed response. Instead, it returns a 500 Internal Server Error, which indicates the method is not being validated at the routing level before reaching business logic that fails unexpectedly.

## Steps to Reproduce
1. Send a DELETE request to the users collection endpoint `/users`.
2. Observe the response status code and body.

## Actual Outcome
The API returns a 500 status code with the body:
```json
{
    "message": "Internal server error"
}
```

## Expected Outcome
The API should return a 405 status code with a body indicating the method is not allowed, for example:
```json
{
    "message": "Method Not Allowed"
}
```

## Sample Payload
```
DELETE /users
```

dev, prod
