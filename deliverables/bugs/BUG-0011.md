# Deleting a non-existing user returns 401 instead of 404

## Description
When a delete request is sent for an email address that does not correspond to any existing user, the API is expected to return a 404 status code with a "User not found" message. Instead, the API returns a 401 status code with an "Authentication required" message, indicating that the authorization check is failing or being evaluated ahead of the "user not found" check on this environment.

## Steps to Reproduce
1. Send a DELETE request to an email endpoint that does not correspond to any existing user, using valid credentials.
2. Observe the response status code and body.

## Actual Outcome
The API returns a 401 status code with the body:
```json
{
    "message": "Authentication required"
}
```

## Expected Outcome
The API should return a 404 status code with a body indicating the user was not found, for example:
```json
{
    "message": "User not found"
}
```

## Sample Payload
`DELETE /users/non.existing.emailAnnabelle.Pollich@hotmail.com`
```
Authorization: <valid token>
```
