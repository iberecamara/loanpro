# Updating a user with PATCH returns 500 instead of 405

## Description
The API does not support partial updates through the PATCH method for a single user resource. Sending a PATCH request to an existing user is expected to be rejected with a 405 Method Not Allowed response. Instead, the API returns a 500 Internal Server Error, indicating the method is not validated before reaching business logic that fails unexpectedly.

## Steps to Reproduce
1. Seed a user in the system so an existing record is available.
2. Send a PATCH request to that user's email endpoint with updated name, email, and age values.
3. Observe the response status code and body.

## Actual Outcome
The API returns a 500 status code with the body:
```json
{
    "message": "Internal server error"
}
```

## Expected Outcome
The API should return a 405 status code with a body indicating the method is not allowed, for example:
```json
{
    "message": "Method Not Allowed"
}
```

## Sample Payload
`PATCH /users/Kenya74@gmail.com`
```json
{
    "name": "Teri Casper-Ratke",
    "email": "Marlen.Cartwright@hotmail.com",
    "age": 120
}
```
