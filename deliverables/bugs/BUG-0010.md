# Deleting an existing user with valid credentials returns 401 instead of 204

## Description
When deleting an existing user with valid credentials, the request is expected to succeed with a 204 No Content response. Instead, the request fails with a 401 status code indicating authentication is required, meaning valid delete requests are being rejected by the authorization check on this environment.

## Steps to Reproduce
1. Seed a user in the system so an existing record is available to delete.
2. Send a DELETE request to that user's email endpoint using valid credentials.
3. Observe the response status code.

## Actual Outcome
The API returns a 401 status code, indicating the delete request was not authenticated.

## Expected Outcome
The API should return a 204 status code with no content, indicating the user was deleted successfully.

## Sample Payload
`DELETE /users/Natasha.Weber86@gmail.com`
```
Authorization: <valid token used by the automated suite>
```
