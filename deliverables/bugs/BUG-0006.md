# Getting a non-existing user by email returns 500 instead of 404

## Description
When a request is made to find a user by an email address that does not exist in the system, the API is expected to return a 404 status code with a message indicating the user was not found. Instead, the API returns a 500 Internal Server Error, suggesting the "user not found" case is not handled and instead causes an unhandled exception.

## Steps to Reproduce
1. Seed a user in the system so at least one valid record exists.
2. Send a GET request to find a user using an email address that does not correspond to any existing user.
3. Observe the response status code and body.

## Actual Outcome
The API returns a 500 status code with the body:
```json
{
    "message": "Internal server error"
}
```

## Expected Outcome
The API should return a 404 status code with a body indicating the user was not found, for example:
```json
{
    "message": "User not found"
}
```

## Sample Payload
```
GET /users/invalid_Jessie.Dickens@hotmail.com
```
