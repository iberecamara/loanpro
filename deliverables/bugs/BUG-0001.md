# Get All Users fails when the request URL has a trailing slash

## Description
The Get All Users endpoint does not respond successfully when the request is sent with a trailing slash at the end of the path. Instead of returning the list of users with a 200 status code, the request fails and the response is not truthy, which breaks any client that appends a trailing slash to the endpoint path.

## Steps to Reproduce
1. Send a GET request to the users endpoint using a trailing slash at the end of the path, for example `/users/`.
2. Observe the response status and body returned by the API.

## Actual Outcome
The API returns a 500 status code with the body:
```json
{
    "message": "Internal server error"
}
```

## Expected Outcome
The API should treat the trailing slash as equivalent to the path without it and return a 200 status code with the list of users in the response body.

## Sample Payload
```
GET /users/
```

## Environments
dev, prod
