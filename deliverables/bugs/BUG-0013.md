# Deleting an existing user with an invalid Authorization header succeeds instead of being rejected

## Description
When a delete request is sent for an existing user with an invalid Authorization header, the API is expected to reject the request with a 401 status code and an "Authentication required" message. Instead, the API processes the request and deletes the user, returning a 204 status code, meaning the delete endpoint does not properly validate the Authorization header value.

## Steps to Reproduce
1. Seed a user in the system so an existing record is available to delete.
2. Send a DELETE request to that user's email endpoint using an invalid or incorrect Authorization header value.
3. Observe the response status code.

## Actual Outcome
The API returns a 204 status code and the user is deleted, even though the Authorization header provided was invalid.

## Expected Outcome
The API should return a 401 status code with a body indicating authentication is required, for example:
```json
{
    "message": "Authentication required"
}
```

## Sample Payload
`DELETE /users/Hazel.Tromp@yahoo.com`
```
Authorization: Bearer invalid-token
```
